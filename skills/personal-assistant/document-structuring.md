# Document Structuring

## Purpose
Turn rough material into a clean, readable document.

## Focus
- audience
- purpose
- section flow
- concise language
- clear next step

## Rule
If the reader cannot scan it quickly, the structure is still wrong.

## Primary Tooling
- Manual: `~/workspace/tools/manual/google.md`
- Script: `~/workspace/tools/general/scripts/google_workspace.py`
- Claude-native structuring is still primary for thinking work

## Usage Path
Use Claude-native structuring first. If output needs to be written to Google Docs, read `tools/manual/google.md` first, then use `google_workspace.py`.
