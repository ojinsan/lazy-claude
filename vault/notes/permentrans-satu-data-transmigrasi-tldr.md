# TLDR — Draft Permentrans Satu Data Kementerian Transmigrasi (v1, 06-05-26)

Draft Peraturan Menteri Transmigrasi: kebijakan **Satu Data Kementerian Transmigrasi** (Satu Data Transmigrasi). Implementasi PerPres 39/2019 (Satu Data Indonesia / SDI) di tingkat Kementerian Transmigrasi. **117 pasal, 16 BAB.**

Dasar hukum utama: PerPres 39/2019 (SDI), PerPres 95/2018 + 132/2022 (SPBE), PermenPPN 16/17/18 2020, Permenkominfo 1/2023 (interop), PerBSSN 4/2021 (keamanan), PermenPANRB 5/2020 (manajemen risiko SPBE), PerPres 172/2024 (Kementerian Transmigrasi), Permentrans 1/2024 (OTK).

---

## BAB I — Ketentuan Umum

### Definisi penting (Pasal 1)
- **Satu Data Indonesia (SDI)** — kebijakan tata kelola data pemerintah agar akurat, mutakhir, terpadu, akuntabel, mudah diakses & dibagipakaikan via Standar Data, Metadata, Interop, Kode Referensi, Data Induk.
- **Satu Data Transmigrasi** — kebijakan tata kelola data di Kementerian Transmigrasi sesuai prinsip SDI.
- **Data** — catatan fakta (angka, simbol, gambar, peta, suara, dst).
- **Data Statistik** — angka karakteristik populasi via pengumpulan, pengolahan, penyajian, analisis.
- **Data Geospasial** — data lokasi geografis, dimensi, karakteristik objek alam/buatan.
- **Standar Data** — standar berisi konsep, definisi, klasifikasi, ukuran, satuan.
- **Metadata** — info struktur+format baku untuk gambarkan & jelaskan data.
- **Interoperabilitas Data** — kemampuan data dibagipakaikan antar sistem.
- **Kode Referensi** — karakter unik sebagai rujukan identitas data.
- **Data Induk** — data objek proses bisnis pemerintah untuk penggunaan bersama.
- **Data Referensi** — data acuan bersama dalam penyelenggaraan data.
- **Data Prioritas** — data terpilih dari daftar data tahun berikutnya, disepakati Forum SDI.
- **Portal SDI / Portal Satu Data Transmigrasi** — media berbagi pakai data nasional / Kementerian.
- **Walidata** — unit yg kumpulkan, periksa, kelola, sebarluaskan data.
- **Produsen Data** — unit yg hasilkan data sesuai kewenangan.
- **Pembina Data** — instansi pusat yg bina data sesuai UU (BPS untuk statistik, BIG untuk geospasial).
- **Forum Satu Data Transmigrasi** — wadah komunikasi/koordinasi Walidata + Produsen + stakeholder.
- **Interoperabilitas Sistem** — sistem elektronik bisa saling komunikasi & tukar data.
- **SPBE** — Sistem Pemerintahan Berbasis Elektronik.
- **Keamanan Informasi** — lindungi kerahasiaan, keutuhan, ketersediaan info.
- **Manajemen Risiko** — pendekatan sistematis identifikasi/analisis/evaluasi/kendalikan risiko.

### Prinsip (Pasal 2)
- **Mandatori 5 (dari SDI):** Standar Data, Metadata, Interoperabilitas Data, Kode Referensi, Data Induk.
- **Plus 7:** akurasi, keterpaduan, kemutakhiran, keterbukaan, akuntabilitas, keamanan informasi, kemudahan akses & berbagi pakai.

### Posisi (Pasal 3)
Bagian dari SPBE. Terintegrasi dengan: SDI + arsitektur SPBE nasional.

### Cakupan data (Pasal 4)
- Jenis: data sektoral transmigrasi, pendukung, lainnya.
- Bentuk: Statistik, Geospasial, lainnya sesuai perkembangan teknologi.

## BAB II — Maksud, Tujuan, Ruang Lingkup

- **Maksud (Pasal 5):** pedoman tata kelola data Kementerian utk perencanaan, pelaksanaan, evaluasi, pengendalian pembangunan transmigrasi.
- **Tujuan (Pasal 6):** acuan tata kelola; data berkualitas; basis kebijakan; perbaiki perencanaan; transparansi; integrasi antar unit & antar instansi; dukung Sistem Statistik Nasional + SPBE.
- **Ruang lingkup pengaturan (Pasal 7):** penyelenggara; penyelenggaraan; standar+metadata; interop; sistem+arsitektur; keamanan; manajemen risiko; portal; manajemen akses; partisipasi & kerja sama; pendanaan.
- **Pelaksanaan (Pasal 8):** terpadu lintas unit; mengacu SDI; integrasi SPBE; perhatikan kebutuhan lintas sektor & lintas wilayah.
- **Cakupan data (Pasal 9):** pembangunan & pengembangan kawasan transmigrasi; ekonomi & pemberdayaan masyarakat; SDM transmigrasi; perencanaan/pelaksanaan/evaluasi program; data pendukung lain. Rincian ditetapkan Menteri.

## BAB III — Penyelenggara (3 Aktor)

### Walidata (Pasal 11–13)
- Unit kerja bidang pengelolaan data & informasi. **Ditetapkan Menteri.**
- Tugas: kumpulkan dari Produsen; periksa kesesuaian SDI; evaluasi kualitas; kelola terintegrasi; sebarluaskan data/metadata/kode/data induk; koordinasi daftar data + Data Prioritas; bantu Pembina Data bina Produsen; koordinasi dgn Pembina Data + Forum SDI.
- Wewenang: minta data+metadata; verifikasi & validasi; kembalikan data tidak sesuai; tetapkan mekanisme pengelolaan & penyebarluasan; usulkan pembatasan akses.
- **Tanggung jawab ke Menteri via Sekretaris Jenderal.**

### Produsen Data (Pasal 14–16)
- Unit kerja yg hasilkan data. Ditetapkan Menteri.
- Tugas: hasilkan data; susun & terapkan Standar Data + Metadata; jamin kualitas; kirim ke Walidata; beri masukan ke Pembina Data; perbaiki data berdasarkan pemeriksaan.
- Wewenang: kelola data; usulkan Data Prioritas; usulkan pembatasan akses ke Walidata.
- Bertanggung jawab atas keakuratan, kemutakhiran, konsistensi data.

### Forum Satu Data Transmigrasi (Pasal 17–19)
- Anggota: Walidata, Produsen Data, stakeholder lain sesuai kebutuhan.
- Tugas: koordinasi daftar data; tetapkan Data Prioritas; susun rencana aksi; bahas pembatasan akses; selesaikan masalah; rekomendasi kebijakan teknis.
- **Dikoordinasi Sekretaris Jenderal.** Pertemuan **min 1x/tahun.**
- Bisa libatkan: K/L lain, pemda, perguruan tinggi, lembaga penelitian, pihak relevan lain.

## BAB IV — Penyelenggaraan (Siklus 4 Langkah)

### Siklus (Pasal 20)
1. Perencanaan Data
2. Pengumpulan Data
3. Pemeriksaan Data
4. Penyebarluasan Data

### Perencanaan (Pasal 21–24)
- Walidata + Produsen via Forum.
- Output: daftar data tahun berikutnya + Data Prioritas + rencana aksi.
- **Daftar data (Pasal 22):** hindari duplikasi; berdasar arsitektur SPBE, kebutuhan kebijakan, kesepakatan Forum, rekomendasi Pembina Data. Memuat: jenis data, Produsen, jadwal rilis/pemutakhiran. **Dasar perencanaan + penganggaran Kementerian.**
- **Data Prioritas (Pasal 23):** Produsen usul → Walidata → Forum. Kriteria: dukung prioritas pembangunan nasional / SDGs / kebutuhan mendesak.
- **Rencana aksi (Pasal 24):** SDM, kebijakan & juknis, kegiatan kumpul/periksa/sebar, kegiatan pendukung. **Ditetapkan Menteri.**

### Pengumpulan (Pasal 25–26)
- Sesuai Standar Data, daftar data, jadwal.
- **Wajib disertai Metadata.** Kirim ke Walidata.
- Perhatikan: kualitas, konsistensi, keterpaduan, keamanan. Bisa pakai sistem elektronik (SPBE).

### Pemeriksaan (Pasal 27–29)
- Walidata cek: kesesuaian Standar Data, kelengkapan Metadata, kualitas/konsistensi, kesesuaian SDI.
- Tidak sesuai → kembalikan; Produsen **wajib perbaiki**.
- **Data Prioritas:** Walidata koordinasi Pembina Data utk pemeriksaan lanjutan; jika gagal → kembalikan ke Walidata → Produsen.

### Penyebarluasan (Pasal 30–32)
- Aktivitas: pemberian akses, pendistribusian, pertukaran.
- Hanya data yg penuhi prinsip SDI.
- Channel: Portal SDI, Portal Satu Data Transmigrasi, media lain sesuai UU.
- Perhatikan: keamanan, klasifikasi, peraturan UU.
- **Penanganan masalah data sudah rilis (Pasal 32):** Walidata + pihak terkait. Masalah meliputi: data tidak sesuai standar; tidak bisa diakses; rusak / risiko keamanan; dalam peninjauan.

## BAB V — Standar Data & Metadata

### Umum (Pasal 33)
Wajib penuhi Standar Data + Metadata. Tujuan: keseragaman definisi, keterbandingan, kualitas, integrasi & interop.

### Standar Data (Pasal 34–35)
- Memuat: konsep, definisi, klasifikasi, ukuran, satuan.
- Disusun Produsen, **ditetapkan Menteri** via Forum.
- Selaras: Standar Data Statistik (BPS) + Standar Data Geospasial (BIG) + UU lain.
- Kalau belum ada standar pembina → Produsen susun standar **sementara**, ditetapkan Menteri.

### Metadata (Pasal 36–37)
- Wajib lengkap: identitas, definisi, sumber, metode pengumpulan, waktu kumpul/pemutakhiran, struktur+format, info lain.
- Disusun Produsen, dikirim Walidata; Walidata periksa; metadata sesuai jadi dasar pengelolaan + sebarluaskan.

### Kode Referensi & Data Induk (Pasal 38–39)
- Untuk konsistensi, hindari duplikasi, integrasi antar sistem.
- Mengacu ke SDI; jika belum tersedia → **Menteri tetapkan kode/data induk sektoral.**

### Pengelolaan (Pasal 40–41)
- Walidata koordinasi pengumpulan/dokumentasi standar, kelola metadata, pemutakhiran, sebarluaskan.
- **Evaluasi min 1x/tahun.**

---

## BAB VI — Interoperabilitas Data

### Umum (Pasal 42)
Prinsip: keterpaduan, keterbukaan, keamanan, efisiensi, keberlanjutan.

### Penyelenggaraan (Pasal 43–44)
- Via **Layanan Interoperabilitas Data (LID).**
- Cakupan: nasional via sistem penghubung layanan + internal Kementerian.
- Wajib: teknologi terbuka, **berbasis API**, terdokumentasi, jamin keamanan, dapat dipantau & dievaluasi.
- Dukung integrasi lintas sistem internal + dgn instansi lain.

### Katalog Layanan (Pasal 45–46)
- Setiap LID **wajib didaftar di katalog layanan.**
- Memuat: nama layanan, penyedia, jenis data, spesifikasi teknis, mekanisme akses.
- Dikoordinasi Walidata; **terintegrasi katalog nasional.**

### Akses & Pemanfaatan (Pasal 47–48)
- Akses **terbuka** (no approval) atau **terbatas** (butuh approval penyedia).
- Pengguna wajib: pakai sesuai UU, jaga keamanan, bertanggung jawab.

### Pengujian & Monev (Pasal 49–50)
- **Wajib uji sebelum live:** uji fungsi + uji keamanan + uji kinerja.
- Walidata monev **min 1x/tahun.** Bisa rekomendasi perbaikan **atau hentikan layanan** yg tidak penuhi ketentuan.

---

## BAB VII — Sistem & Arsitektur Data

### Umum (Pasal 51)
Sistem elektronik terintegrasi, sesuai SPBE. Dukung: pengelolaan end-to-end, integrasi, interop, layanan berbasis data.

### Arsitektur (Pasal 52–54) — 5 lapis
1. Bisnis
2. Data
3. Aplikasi
4. Infrastruktur
5. Keamanan

- **Arsitektur data:** struktur+relasi data, integrasi lintas unit, kualitas+konsistensi. Selaras arsitektur SPBE nasional + SDI.
- **Arsitektur aplikasi:** dukung layanan berbasis data, hindari duplikasi, efisiensi. Prinsip: **interop, modular, skalabel, teknologi terbuka.**

### Integrasi (Pasal 55–56)
- Via LID + sistem penghubung layanan.
- Sistem wajib: terintegrasi, dukung interop, pakai standar data+metadata. Hindari duplikasi fungsi+data.

### Pengelolaan (Pasal 57–58)
- **SDLC 7 langkah:** analisis kebutuhan → perencanaan → perancangan → pembangunan → pengujian → implementasi → pemeliharaan.
- Sistem wajib: terdokumentasi, aman+andal, dukung integrasi, penuhi standar keamanan info.
- Dokumentasi minimal: desain sistem, spesifikasi teknis, dokumentasi data, dokumentasi integrasi.

### Infrastruktur (Pasal 59–60)
- Pakai **Pusat Data Nasional (PDN)** atau yang ditetapkan UU.
- Penyimpanan **terpusat & terintegrasi** → ketersediaan, keamanan, akses.
- Walidata koordinasi pengelolaan; perhatikan efisiensi sumber daya, keamanan+keandalan, keberlanjutan.

---

## BAB VIII — Keamanan Data & Informasi

### Prinsip (Pasal 61–62)
- **CIA**: kerahasiaan (confidentiality), keutuhan (integrity), ketersediaan (availability).
- **Plus**: keaslian (authenticity), kenirsangkalan (non-repudiation), keterlacakan (accountability).

### Klasifikasi Data (Pasal 63–64) — 3 tingkat
1. **Terbuka**
2. **Terbatas**
3. **Rahasia**

Ditetapkan Walidata + Produsen. Drive: pemberian akses, pengamanan, penyebarluasan.

### Pengamanan (Pasal 65–66)
- Cegah: akses tidak sah, perubahan tak sah, kebocoran, gangguan sistem.
- Mekanisme: **pengendalian akses, enkripsi, pengamanan jaringan, lainnya.**
- Sistem wajib: **autentikasi+otorisasi, log aktivitas, sistem pemantauan keamanan.**

### Akses & Penggunaan (Pasal 67–68)
- Akses sesuai klasifikasi + kewenangan + tujuan.
- Pengguna wajib: jaga rahasia, pakai sesuai tujuan sah, tidak salahgunakan. Pelanggaran → sanksi UU.

### Insiden (Pasal 69–70)
- Wajib lapor Walidata.
- Penanganan: **identifikasi → tangani+pulihkan → laporkan+evaluasi.**
- Hasil utk peningkatan keamanan ke depan.

## BAB IX — Manajemen Risiko Data & SPBE

### Tujuan (Pasal 71)
Identifikasi risiko, minimize dampak, andalkan sistem+data, dukung keputusan berbasis risiko.

### Prinsip (Pasal 72)
Terintegrasi, terstruktur+komprehensif, adaptif, berbasis data, berkelanjutan.

### Proses (Pasal 73) — 6 langkah
1. Komunikasi & konsultasi
2. Penetapan konteks
3. Penilaian risiko
4. Penanganan risiko
5. Monev
6. Pelaporan

### Penilaian (Pasal 74)
- Sub-langkah: identifikasi → analisis → evaluasi.
- Objek: data, sistem+aplikasi, infrastruktur, keamanan info, proses bisnis.

### Penanganan (Pasal 75–76) — 4 strategi
- **Mitigasi / penghindaran / pemindahan / penerimaan.**
- Risiko residual tetap dipantau hingga tingkat dapat diterima.

### Monev & Pelaporan (Pasal 77–78)
Berkala / insidental. Pastikan efektivitas, deteksi risiko baru, tingkatkan kualitas. Wajib didokumentasi & dilaporkan → bahan keputusan + perbaikan.

### Tanggung Jawab (Pasal 79)
- **Walidata** koordinasi.
- **Produsen** kelola risiko data yg dihasilkan.
- **Setiap unit** bertanggung jawab risiko di lingkup tugasnya.

---

## BAB X — Portal Satu Data Transmigrasi

### Umum (Pasal 80)
Media berbagi pakai data Kementerian. Tujuan: akses, integrasi, transparansi, pemanfaatan.

### Pengelolaan (Pasal 81–82)
- Dikelola **Walidata.**
- Cakupan: kumpul+kelola data, kelola metadata, akses, pemeliharaan, pengembangan fitur.
- Pengembangan sesuai UU SPBE. Prinsip portal: interop, keamanan, kemudahan akses, keterpaduan, keberlanjutan.

### Integrasi (Pasal 83–84)
- **Wajib terintegrasi Portal SDI** via LID + sistem penghubung layanan.
- Bisa terintegrasi: sistem internal Kementerian, sistem K/L lain, sistem pemda.

### Penyediaan & Pemanfaatan (Pasal 85–86)
- Walidata sediakan data via portal.
- Data wajib: penuhi Standar Data, lengkap Metadata, lulus pemeriksaan.
- Akses sesuai klasifikasi.

### Pemeliharaan & Evaluasi (Pasal 87–88)
- Pemeliharaan berkala: sistem, pemutakhiran data, peningkatan keamanan.
- Evaluasi berkala: kualitas layanan, ketersediaan+keandalan, pemanfaatan.

---

## BAB XI — Hak Akses & Pembatasan Data

### Umum (Pasal 89)
Akses sesuai UU. Perhatikan klasifikasi, kewenangan, tujuan.

### Pemberian Akses (Pasal 90–92)
- **Akses terbuka** = data publik, tanpa persetujuan khusus, tetap perhatikan perlindungan data.
- **Akses terbatas** = data sensitivitas tertentu, butuh permohonan + persetujuan Walidata/Produsen. Mekanisme diatur Walidata.

### Pembatasan (Pasal 93–94)
- Kriteria: rahasia, ancaman keamanan negara, langgar UU, sensitivitas tinggi.
- Walidata/Produsen usul → Forum bahas → **ditetapkan Menteri.**

### Penggunaan (Pasal 95–96)
- Wajib sesuai tujuan sah.
- Dilarang: salahguna, distribusi tanpa izin, tujuan bertentangan UU.
- Pelanggaran → sanksi UU.

---

## BAB XII — Partisipasi & Kerja Sama

### Partisipasi (Pasal 97–99)
- Tujuan: tingkatkan kualitas, dukung integrasi, perluas pemanfaatan.
- Bentuk: penyediaan data, pengolahan, validasi, pemanfaatan, pengembangan sistem+teknologi.
- Pihak: K/L, pemda, badan hukum publik, perguruan tinggi, lembaga riset, dunia usaha, masyarakat, lainnya.

### Kerja Sama (Pasal 100–101)
- Untuk: tukar data, integrasi sistem, kapasitas SDM, teknologi+inovasi, lainnya.
- Mitra: K/L, pemda, perguruan tinggi, lembaga riset, badan usaha, **lembaga internasional.**

### Pengelolaan (Pasal 102)
Walidata koordinasi. Perhatikan keamanan, perlindungan data, kepentingan nasional, UU.

---

## BAB XIII — Pelaporan & Evaluasi

### Pelaporan (Pasal 103–105)
- **Produsen → Walidata: min 2x/tahun.**
- **Walidata → Menteri: laporan tahunan** memuat: pelaksanaan pengelolaan, kualitas, pemanfaatan, kendala+masalah, rekomendasi perbaikan.

### Evaluasi (Pasal 106–108)
- Walidata, **min 1x/tahun.**
- Cakupan: kesesuaian pelaksanaan, kualitas data, tingkat integrasi, pemanfaatan, efektivitas sistem+proses.
- Hasil utk: perbaikan, peningkatan kualitas, pengembangan kebijakan+sistem; bahan rencana aksi.

### Tindak Lanjut (Pasal 109)
Walidata + Produsen wajib tindak lanjut perbaikan secara terencana & terukur.

## BAB XIV — Pendanaan

- Sumber (Pasal 110): **APBN Kementerian** + sumber sah lain tidak mengikat.
- Penggunaan (Pasal 111): tata kelola, sistem+infra, portal, SDM, kegiatan pendukung.
- Prinsip (Pasal 112): efisiensi, efektivitas, transparansi, akuntabilitas. Sesuai UU keuangan negara.

## BAB XV — Ketentuan Peralihan

- **Pasal 113:** data, sistem, kerja sama existing tetap berlaku selama tidak bertentangan; **wajib disesuaikan.**
- **Pasal 114:** penyesuaian **paling lama 2 tahun** sejak diundangkan. Cakupan:
  1. kelembagaan
  2. proses pengelolaan data
  3. Standar Data + Metadata
  4. sistem + integrasi data
  5. kebijakan + prosedur
- **Pasal 115:** Walidata koordinasi transisi; Produsen wajib dukung.

## Cheatsheet — Hafalkan ini

### 5 prinsip mandatori SDI
Standar Data | Metadata | Interoperabilitas Data | Kode Referensi | Data Induk

### 7 prinsip plus
Akurasi | Keterpaduan | Kemutakhiran | Keterbukaan | Akuntabilitas | Keamanan Informasi | Kemudahan akses & berbagi pakai

### 3 penyelenggara
Walidata (gatekeeper, ditetapkan Menteri, jawab via Sekjen) | Produsen Data (penghasil) | Forum (koordinasi, dipimpin Sekjen, min 1x/thn)

### 4 langkah siklus
Perencanaan → Pengumpulan → Pemeriksaan → Penyebarluasan

### 3 klasifikasi
Terbuka | Terbatas | Rahasia

### 6 prinsip keamanan
CIA (Confidentiality, Integrity, Availability) + Authenticity + Non-repudiation + Accountability

### 6 langkah manajemen risiko
Komunikasi → Konteks → Penilaian → Penanganan → Monev → Pelaporan

### 4 strategi penanganan risiko
Mitigasi | Avoidance | Transfer | Acceptance

### 5 lapis arsitektur
Bisnis | Data | Aplikasi | Infrastruktur | Keamanan

### 7 langkah SDLC
Analisis → Perencanaan → Perancangan → Pembangunan → Pengujian → Implementasi → Pemeliharaan

### Cadence
- Forum: min 1x/thn
- Evaluasi standar+metadata: min 1x/thn
- Monev LID: min 1x/thn
- Evaluasi penyelenggaraan SDT: min 1x/thn
- Produsen → Walidata: min 2x/thn
- Walidata → Menteri: tahunan

Prioritas pembangunan nasional | SDGs | Kebutuhan mendesak

### LID wajib
Teknologi terbuka | API-based | Terdokumentasi | Aman | Monitorable | Terdaftar di katalog layanan

### Transisi
**2 tahun** sejak diundangkan utk penyesuaian penuh.

Walidata | Produsen Data | Standar Data | Rencana Aksi | Pembatasan Akses Data | Kode Referensi/Data Induk sektoral
