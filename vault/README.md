# Trader Vault

Obsidian vault for Boss O. Open this folder in Obsidian.app (or any MD editor).

JSON in `data/` is the machine source of truth. The MD in every other folder is the human record. Code writes both. Never hand-edit `data/*.json`.

## Folders

| Dir | Owner | Purpose |
|-----|-------|---------|
| `daily/` | Boss O + Claude | One MD per day. Free-form at top, Claude appends under `## Auto-Appended` via `journal.append_daily_layer_section()`. One scroll = one day of context. |
| `thesis/` | Claude writes, Boss O audits | One MD per ticker with an active thesis. Append-only Review Log via `journal.append_thesis_review()`. Template: `.templates/thesis.md`. |
| `themes/` | L1 | One MD per active narrative theme. Frontmatter (`theme`, `status`, `sector`, `related_tickers`). Template: `.templates/theme.md`. |
| `journal/` | Claude (L0 Step 6) | Daily self-reviews. What worked, what missed, what to change. |
| `lessons/` | `log_lesson_v2` | Dated + slugged MD per lesson with frontmatter and Obsidian tags. Linked to tickers and thesis. |
| `layer-output/{l0..l5}/` | Layer playbooks | Raw per-run outputs archived per date. Regeneratable — gitignored. |
| `reviews/{weekly,monthly}/` | `generate_weekly_review` / `generate_monthly_review` | Auto-generated summary MDs. Regeneratable — gitignored. |
| `data/` | `journal.py`, `portfolio_health.py`, `vault_sync.py` | Machine-read JSON. DO NOT EDIT BY HAND. Gitignored. |
| `.templates/` | maintained manually | Note scaffolds — not tracked in Obsidian search due to dot prefix. |

## Conventions

### Wikilinks
- Ticker → `[[ANTM]]` (linked to `thesis/ANTM.md` when active)
- Theme → `[[nickel-ev-rebound]]` (linked to `themes/nickel-ev-rebound.md`)
- Lesson backlink → `[[2026-04-16-early-exit-on-noise-1]]`

### Tags (inline, space-separated)
- `#sector/nickel` `#sector/banking` `#sector/energy` `#sector/property` `#sector/consumer`
- `#setup/accumulation` `#setup/breakout` `#setup/shakeout-recovery` `#setup/swing`
- `#layer/l0` … `#layer/l5`
- `#severity/high` `#severity/medium` `#severity/low` — lessons only
- `#category/entry_timing` `#category/exit_timing` `#category/thesis_quality` `#category/sizing` `#category/psychology` `#category/missed_trade` `#category/portfolio`
- `#pattern/<slug>` — matches `pattern_tag` in lesson JSON; enables `detect_recurring_mistakes` grouping

### Frontmatter

Thesis:
```yaml
ticker: ANTM
created: 2026-04-16
layer_origin: L2
status: active        # active | closed | archived
setup: accumulation
related_themes: [[nickel-ev-rebound]]
related_lessons: []
```

Theme:
```yaml
theme: nickel-ev-rebound
created: 2026-04-16
status: active

related_tickers: [ANTM, INCO, MBMA]
```

Lesson:
```yaml

tickers: [ANTM]

```

## Append-Only Rule

Files in `thesis/`, `themes/`, and daily notes grow over time. Code paths NEVER overwrite:
- `append_thesis_review(ticker, layer, note)` — new line under `## Review Log`
- `append_daily_layer_section(layer, content)` — new block under `## Auto-Appended`
- Manual edits in Obsidian are always fine — code won't touch unrelated sections.

Status flips (thesis closed, theme invalidated) are done by hand in Obsidian — no auto-rewrite from code.

## Obsidian Search Examples

- All active thesis → `status: active path:thesis/`
- High-severity lessons last 30d → `#severity/high created:>2026-03-15`
- Everything tied to ANTM → click `[[ANTM]]` in any note, see backlinks pane
- Recurring patterns → search `#pattern/early-exit-on-noise`
- This week's layer output → `path:layer-output/ created:>2026-04-13`

## Gitignored (regenerated by code)

- `data/*.json`
- `layer-output/**`
- `reviews/**`

These recreate on next run. Don't panic if Obsidian shows them missing after a fresh clone — run the cron or call the relevant function.

## Templates

To create a new thesis: copy `.templates/thesis.md` → `thesis/<TICKER>.md`, fill frontmatter + sections. Or call `append_thesis_review()` which bootstraps from the template.
